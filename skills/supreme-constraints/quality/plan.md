# 规划质量约束

# 一、任务拆分原则

按架构文档拆分原子任务，每个任务必须满足：
- 由单一 agent 完成
- 有明确的产出物
- 有明确的验收标准
- **必须尽可能拆分到最细粒度**

# 二、任务依赖原则
**代码编写任务**：不依赖其他任务，或依赖其他代码编写任务
**代码审查任务**：依赖对应的代码编写任务
**集成测试任务**：不依赖其他任务，或依赖若干个代码编写任务及对应的代码审查任务
**集测审查任务**：依赖对应的集成测试任务

依赖关系用途：任务失败时定位直接和间接依赖于失败任务的所有任务

# 三、执行者分配原则
**代码编写任务**：Coder
**代码审查任务**：Code Reviewer
**集成测试任务**：Tester
**集测审查任务**：Test Reviewer
