# 规划质量约束

# 一、任务拆分原则

按架构文档拆分原子任务，每个任务必须满足：
- 由单一 agent 完成
- 有明确的产出物
- 有明确的验收标准
- **必须尽可能拆分到最细粒度**

# 二、任务依赖原则
**代码编写任务**：或依赖其他代码编写任务的产出
**代码审查任务**：依赖对应的代码编写任务完成
**集成测试任务**：依赖所有代码编写任务完成 + 所有代码审查任务完成
**集测审查任务**：依赖对应的集成测试任务完成

依赖关系用途：任务失败时定位直接和间接依赖于失败任务的所有任务。

# 三、执行者分配原则
**代码编写任务**：分配给 Coder
**代码审查任务**：分配给 Code Reviewer
**集成测试任务**：分配给 Tester
**集测审查任务**：分配给 Test Reviewer

# 四、规划文档格式
**绝对不可添加任何该格式未包含的内容**
**绝对不可修改该格式，只能精确地按照该格式填空**
