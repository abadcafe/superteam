# 规划审查报告约束

## 格式
```markdown
# 规划审查报告: [与规划文档标题保持一致]

审查时间: [时间]

## 审查结果

### 🔴 严重问题（必须修复）

#### 问题 N: [问题标题]
**位置**: [规划文档章节]
**描述**: [问题描述]
**设计依据**: [引用架构文档相关内容（如有，否则填无）]
**备注**: [说明为何属于严重问题]

### 🟡 普通问题（建议修复）

#### 问题 M: [问题标题]
**位置**: [规划文档章节]
**描述**: [问题描述]
**设计依据**: [引用架构文档相关内容（如有，否则填无）]
**备注**: [说明为何属于普通问题]

## 总结
- 严重问题: N 个
- 普通问题: M 个

## 结论
- [ ] 有问题，需修复
- [ ] 无问题
```

### 问题分级

#### 🔴 严重问题（必须修复）
定义：影响任务拆分粒度、任务依赖准确性、任务执行和任务描述准确性的问题
举例：
- 任务未覆盖架构文档所有功能点
- 任务超出架构文档范围
- 任务描述不清晰或与架构文档不一致
- 任务拆分粒度过大
- 任务依赖关系错误
- 任务类型错误
- 执行者分配错误

#### 🟡 普通问题（建议修复）
定义：其他不影响功能、性能、安全性、稳定性、可运维性和代码质量的问题
