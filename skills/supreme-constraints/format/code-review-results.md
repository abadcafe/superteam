- 接口数: N

### 问题分级标准

#### 🔴 严重问题（必须修复）
定义：影响功能、性能、安全性、稳定性、可运维性和代码质量的问题
举例：
- 功能与设计不符
- 安全漏洞、数据丢失风险、内存泄漏、资源泄漏、性能瓶颈、系统崩溃等
- 代码可读性差、重复代码
- 缺少错误处理
- 单测覆盖率不足或未通过或不可重复执行

#### 🟡 普通问题（必须修复）
定义：不影响功能、性能、安全性、稳定性、可运维性和代码质量的问题
举例：
- 非规范要求的代码风格偏好
- 文档或注释的表述不清晰或有歧义
