# 问题分析报告约束

## 格式
```markdown
# 问题分析报告: [问题标题]

## 分析范围
- 触发任务: task-XXX
- 问题现象: [问题现象描述]
- 分析时间: [时间]

## 问题根源

### 问题类型
- [ ] 功能代码问题
- [ ] 单测代码问题
- [ ] 集测代码问题
- [ ] 配置问题
- [ ] 环境问题

### 根源定位

#### 如果是功能代码问题
- 问题根源任务: task-XXX, task-YYY, ...
- 问题原因: [详细说明为什么是这些任务的问题]
- 受影响的下游任务: [列出所有直接或间接依赖于这些任务的任务]

#### 如果是单测代码问题
- 问题根源任务: task-XXX
- 问题原因: [详细说明单测代码的问题]
- 修复建议: [如何修复单测代码]

#### 如果是集测代码问题
- 问题根源任务: task-XXX
- 问题原因: [详细说明集测代码的问题]
- 修复建议: [如何修复集测代码]

#### 如果是配置问题
- 配置文件: [文件路径]
- 问题原因: [详细说明配置问题]
- 修复建议: [如何修复配置]

#### 如果是环境问题
- 问题原因: [详细说明环境问题]
- 解决建议: [如何解决环境问题]

## 分析过程
[记录分析过程，便于追溯]

## 修复建议
[详细的修复建议]

## 结论
问题根源: [任务编号/配置文件/环境]
建议处理方式: [处理建议]
```
