# 问题分析报告约束

## 格式
```markdown
# 问题分析报告: [问题标题]

## 分析范围
- 触发任务: task-XXX
- 问题现象: [问题现象描述]
- 分析时间: [时间]

## 问题根源

### 问题类型
- [ ] 功能代码问题
- [ ] 单测代码问题
- [ ] 集测代码问题
- [ ] 配置问题
- [ ] 环境问题
- [ ] 架构问题

### 根源定位

- 问题根源任务: [task-XXX, task-YYY, ... 没有则填无]
- 受影响的下游任务: [列出所有直接或间接依赖于这些任务的任务编号，没有则填无]
- 问题原因: [详细说明问题原因]

## 修复建议
[详细的修复建议]

## 分析过程
[记录分析过程，便于追溯]

## 结论
[如果有问题根源任务则为必须修复，否则为跳过]
```

### 问题根源类型

#### 1. 功能代码问题
- 问题根源是一个或多个代码编写任务
- 需要定位到具体的任务编号
- 需要说明为什么是这些任务的问题

#### 2. 集测代码问题
- 问题根源是集成测试任务
- 集测用例本身有问题
- 需要说明具体是什么问题

#### 3. 配置问题
- 问题根源是配置文件
- 需要说明具体是什么配置问题
- 需要建议如何解决

#### 4. 环境问题
- 问题根源是运行环境
- 需要说明具体是什么环境问题
- 需要建议如何解决

#### 5. 架构问题
- 问题根源是架构设计
- 需要说明具体是什么架构问题
- 需要建议如何解决
