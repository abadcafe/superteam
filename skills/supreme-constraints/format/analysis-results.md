# 问题分析报告约束

## 格式
```markdown
# 问题分析报告：[问题标题]

## 分析范围
- 触发任务：task-XXX
- 问题现象：[问题现象描述]
- 分析时间：[时间]

## 问题 N：[问题标题]
- 问题根源任务：[task-XXX, task-YYY, ... 没有则填无]
- 受影响的下游任务：[列出所有直接或间接依赖于这些任务的任务编号，没有则填无]
- 问题类型：[问题类型]
- 问题描述：
[描述]

- 问题原因：
[原因]

- 修复建议：
[修复建议]

- 分析过程：
[记录分析过程，便于追溯]

## 结论
[如果有问题根源任务则为必须修复，否则为跳过]
```

### 问题类型

#### 1. 功能代码问题
- 问题根源是代码编写任务
- 需要说明为什么是这些任务的问题

#### 2. 单测代码问题
- 问题根源是代码编写任务
- 需要说明为什么是这些任务的问题

#### 3. 集测代码问题
- 问题根源是集成测试任务
- 需要说明具体是什么问题

#### 4. 配置问题
- 问题根源是配置文件
- 需要说明具体是什么配置问题

#### 5. 环境问题
- 问题根源是运行环境
- 需要说明具体是什么环境问题

#### 6. 架构问题
- 问题根源是架构设计
- 需要说明具体是什么架构问题
