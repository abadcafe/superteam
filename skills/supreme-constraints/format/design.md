# 架构文档约束

## 格式
```markdown
# 架构文档：[与需求文档标题保持一致]

## 1. 需求概述

[需求简要说明]

### 1.1 需求问题
[描述与决策，没有则填无]

## 2. 技术方案

### 2.1 核心对象及其职责

[对象名称]
- 职责: [职责描述]

[对象名称]
- 职责: [职责描述]

### 2.2 协作关系

[对象之间的依赖和调用关系说明]

### 2.3 模块改动清单
- [模块名]: [新增/修改/删除] - [说明]
- [模块名]: [新增/修改/删除] - [说明]

src/
├── [文件名]
└── [目录名]/
    └── [文件名]

## 3. 内部接口定义（函数级）

### 3.1 [接口名称]

**参数说明**
- [参数名]: [含义和约束]

**返回值说明**
- [返回值含义和可能值]

**行为描述**
[接口的行为说明]

**错误类型**
- [可能的错误类型和处理方式]

## 4. 外部接口定义

### 4.1 [接口名称]

**调用方式**
[url, grpc...]

**参数说明**
- [参数名]: [含义和约束]

**返回值说明**
- [返回值含义和可能值]

**行为描述**
[接口的行为说明]

**错误类型**
- [可能的错误类型和处理方式]

## 5. 行为规范

### 5.1 处理流程
[详细处理流程]

### 5.2 错误处理
[错误处理方式]

### 5.3 边界条件
[需要考虑的边界条件]

## 6. 单测要点列表

- [哪些条件下哪些地方需要验证]
- [单测覆盖率要求100%]

## 7. 集测场景列表

- [在哪些程序运行场景下验证哪些功能点]
- [每个功能点的各种边界条件都要覆盖到]
- [集测覆盖率要求100%]

## 8. 任务列表

### 任务 001: [任务名称]
- **类型**: 代码编写任务 / 代码审查任务 / 集成测试任务 / 集测审查任务
- **依赖**: [依赖的任务编号（如有）]

**任务目标**:
[描述（要做成什么样，而不是要怎么做）]

### 任务 002: [任务名称]
[同上结构]
```

### 任务类型
有且只有以下4种任务：
- 代码编写任务：实现一个或多个功能点的代码和单测
- 代码审查任务：审查功能代码和单测代码
- 集成测试任务：实现一个或多个功能点的集成测试（设计+编写+执行）
- 集测审查任务：审查集成测试代码
