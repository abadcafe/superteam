# 集测审查报告约束

## 格式
```markdown
# 集测审查报告: [任务编号 - 任务名称]

审查时间: [时间]

## 审查范围
任务: [任务编号-任务名称]
变更文件列表: [实际变更的文件列表]

## 审查结果

### 🔴 严重问题（必须修复）

#### 问题 1: [问题标题]
**位置**: [file.py]:[行号]
**描述**: [问题描述]
**设计依据**: [引用架构文档相关内容（如有，否则填无）]
**备注**: [说明为何属于严重问题]

### 🟡 普通问题（必须修复）

#### 问题 1: [问题标题]
**位置**: [file.py]:[行号]
**描述**: [问题描述]
**设计依据**: [引用架构文档相关内容（如有，否则填无）]
**备注**: [说明为何属于普通问题]

## 集测审查

### 覆盖率检查
- 功能点数: N
- 已覆盖: M
- 覆盖率: X%
- 是否达标: ✅/❌

## 总结
- 严重问题: N 个
- 普通问题: M 个

## 结论
- [ ] 有问题，需修复
- [ ] 无问题
```

### 问题分级标准

#### 🔴 严重问题（必须修复）
定义：影响集测质量的问题
举例：
- 集测用例未覆盖架构文档所有功能点
- 集测用例逻辑错误
- 缺少 Type Annotation
- 使用临时脚本（heredoc、内联Python等）
- 集测不可重复执行
- Mock 设计不合理
- 违反黑盒集测原则

#### 🟡 普通问题（建议修复）
定义：不影响集测质量的问题
