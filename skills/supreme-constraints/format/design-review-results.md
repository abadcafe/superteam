# 架构审查报告约束

## 格式
```markdown
# 架构审查报告: [与架构文档标题保持一致]

审查时间: [时间]

## 审查对象

路径: [架构文档路径]

## 审查结果

### 🔴 严重问题（必须修复）

#### 问题 N: [问题标题]
- **位置**: [架构文档章节]
- **描述**: [问题描述]
- **需求依据**: [引用需求文档相关内容（如有，否则填无）]
- **备注**: [说明为何属于严重问题]

### 🟡 普通问题（建议修复）

#### 问题 M: [问题标题]
- **位置**: [架构文档章节]
- **描述**: [问题描述]
- **需求依据**: [引用需求文档相关内容（如有，否则填无）]
- **备注**: [说明为何属于普通问题]

## 总结

- 严重问题: N 个
- 普通问题: M 个

## 结论

- [ ] 有问题，必须修复
- [ ] 无问题
```

### 问题分级标准

#### 🔴 严重问题（必须修复）
定义：影响功能、性能、安全性、稳定性、可运维性和代码质量的问题
举例：
- 设计与需求不符
- 设计存在明显技术缺陷
- 设计存在安全漏洞风险
- 设计存在性能瓶颈风险
- 设计内容违反规范约束要求

#### 🟡 普通问题（建议修复）
定义：其他不影响功能、性能、安全性、稳定性、可运维性和代码质量的问题
